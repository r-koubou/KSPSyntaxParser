§フォルダ構成

/KSPSyntaxParser
    |
    +-- build/                  ant ビルドスクリプト、及びビルドの設定ファイル
    |
    +-- deploy/                 配布用アーカイブファイルの生成先
    |
    +-- lib/                    外部依存のクラスライブラリの置き場所
    |
    +-- src/
    |    |
    |    +-- java/              javaソースファイル置き場（このフォルダを起点にコンパイルされます）
    |    |
    |    +-- javadoc/           javadocにより生成される物以外に同梱するファイルがあればここに置きます
    |    |
    |    +-- package/           このフォルダ以下のファイル・フォルダ構成を配布アーカイブ内のルート位置に配備されます
    |    |                      README.txt などのファイルを置くのに適しています
    |    |
    |    +-- resources/         アプリケーションの jar ファイルに同梱させるファイル・フォルダをここに置きます
    |                           ※必要に応じて build.xml 内の javadoc タスクのオプションを設定してください。
    |
    +-- target/                 ビルド実行のための作業用フォルダ
    |
    +-- build.xml               一連のビルド処理を記述した ant スクリプト
    |
    +-- build.properties        build.xml が参照する変数の定義ファイル
    |
    +-- README.txt              このファイル
    |
    +-- manifest.mf             アプリケーションの jar ファイルに同梱させるマニフェストファイル
    |
    +-- .classpath              統合環境 Eclipse 用のプロジェクトファイル
    |
    +-- .project                統合環境 Eclipse 用のプロジェクトファイル


§ビルド方法

    その１:
    コマンドラインから、build.xml があるフォルダをカレントにし ant を実行します。

    usage:
        ant <ターゲット名>


    ターゲット名として以下の物が build.xml 内で定義されています。

    clean           : ビルド処理で生成された成果物を削除します

    compile         : コンパイルを行います

    jar             : コンパイルされて生成された classファイルをアーカイブし、jarファイルを作成します

    apidoc          : javadoc を生成します

    all             : ビルド処理で生成された成果物を削除した後、配布形式のアーカイブファイルを生成します。

    package         : 配布形式のアーカイブファイルを生成します。

    run             : このプロジェクトのプログラムを実行します。生成された jar ファイルを使用します。
                      コマンドライン引数を指定する場合は build.properties を確認してください。
                      細かいコマンドライン引数の動作チェックなどで引数が不定の場合はこのタスクを使わずに
                      直接 java コマンドから実行してください。


    ※<ターゲット名>を指定しなった場合、 "all" として実行されます。
    ※antを使用する場合、 環境変数 JAVA_HOME が設定されている必要があります


    その２:
    同梱のバッチファイルを使用する

    build.bat       : ant jar と同等のコマンド

    compile.bat     : ant compile と同等のコマンド

    javadoc.bat     : ant apidoc と同等のコマンド

    deploy.bat      : ant package と同等のコマンド

    clean.bat       : ant clean と同等のコマンド

    all.bat         : ant all と同等のコマンド

    run.ant.bat     : ant run と同等のコマンド

    ※上記バッチファイルを使用する場合は、同梱の setenv.bat 内に記載されている環境変数を
      前もって設定してください。

§プロジェクトを Eclipse 上で使用する

    Eclipseにて「ファイル」⇒「インポート」⇒「既存プロジェクトのインポート」の手順で
    このファイルがあるフォルダを指定することで Eclipse 上で使用可能になります。
    Eclipse上でコンパイルされたファイルは「eclipse_compiled」というフォルダに出力されます。

    勿論、Eclipse 上で build.xml を使用したビルドも可能です。
